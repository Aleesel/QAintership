# Bug Report: Отсутствие валидации на отрицательное значение цены при создании товара

## Краткое описание

При создании товара через POST-запрос к `/api/1/item` с отрицательным значением поля `price` API возвращает `200 OK` и успешно создает товар, хотя очевидно, что цена не может быть отрицательной.

## Шаги для воспроизведения

1.  Отправить POST-запрос на `/api/1/item`.
2.  В теле запроса указать JSON, например:
    ```json
    {
      "name": "Тестовый товар с отрицательной ценой",
      "price": -100,
      "sellerId": 12345,
      "statistics": {
                     "likes": 23,
                     "viewCount": 58,
                     "contacts": 3
                     }
    }
    ```
3.  Выполнить DELETE-запрос на `/api/2/item/{id}`, где `{id}` - идентификатор созданного товара.

## Фактический результат

*   POST-запрос: Возвращается статус `200 OK`, товар успешно создается.
*   DELETE-запрос: Возвращается статус `200 OK`, товар успешно удаляется.

## Ожидаемый результат

*   POST-запрос: Должен возвращать статус `400 Bad Request` с соответствующим сообщением об ошибке валидации, указывающим, что `price` не может быть отрицательным.

## Приоритет

Средний (Medium) - фактически ничего не ломает, но может привести к дальнейшим ошибкам

## Дополнительная информация

*   Тест-кейс: `TC-13 — Создание товара с отрицательной ценой`
